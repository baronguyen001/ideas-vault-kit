# 03 - Scale & Unit Economics

## Core question

Can PostPal become a small profitable SaaS without high support or platform costs?

---

## 3.1 Revenue Model

- Type: subscription with a free tier.
- Pricing tiers: $0 / $9 / $19 per month.
- Target ARPU: $11-14 if most paid users are solo builders.

## 3.2 Unit Economics

| Metric | Value | Calculation |
|---|---:|---|
| Revenue / paid user / year | $108-$228 | $9 or $19 monthly |
| COGS / user / year | $6-$30 | Hosting, queue, storage, API overhead |
| Gross margin | 80-95% | Depends on X API cost |
| CAC | $0-$30 | Founder-led content, communities, Product Hunt |
| LTV | $80-$180 | Assumes 8-10 month average retention |
| LTV/CAC | 3-6 | Healthy if organic channel works |
| Payback period | 0-3 months | Mostly self-serve |
| Monthly churn | 5-10% | Creator tools churn when users stop posting |

## 3.3 Scale Ceiling

- One person can support hundreds of users if platform integrations are stable.
- 10x requires better observability, retry tooling, and support templates.
- Biggest bottleneck is API reliability, not onboarding.

## 3.4 Recurring vs One-Time

Revenue is recurring. Break-even can be low:

- $500/mo infrastructure and tools.
- About 56 users on $9/mo or 27 users on $19/mo cover direct costs.
- A meaningful indie target is 1,000 users at $11 ARPU = $11k MRR.

## 3.5 Bull vs Bear

**Bull case**:

Low marginal cost, simple self-serve onboarding, and a clear audience make the economics attractive.

**Bear case**:

If API access becomes expensive or support-heavy, the $9 tier may not work and churn could rise.

---

## Score: 8/10

**Reasoning**:

The economics are clean for a narrow SaaS: low hosting cost, self-serve acquisition, and subscription revenue. Platform API costs are the main unknown.
